Let's bring these features together. It has the advantage, that your private data as well as the passwords to access your data are stored securely. To achieve this, two software components are required, that connect KeePassXC to Cryptomator.

Ralph Plawetzki

The plug-in

Cryptomator has an integrations-api, that allows to extend it. Nevertheless, this API itself had to be extended, because the extensions available for Cryptomator so far had no need to associate to Cryptomator. They are simply available on the underlying OS.

I wrote a library, that works as a connector between Cryptomator and KeePassXC and uses the extended API.

Show info

The back end

A second library is needed. One, that enables an application to talk to KeePassXC and access its capabilities. The Java library accesses KeePassXC via its build-in proxy.

The library I wrote uses the JSON data format to communicate with KeePassXC. The underlying TweetNaCL crypto library provides the crypto_box functionality that is used to set up an encrypted and secure channel for the components to communicate with each other.

It's fast, easy to use and cross-platform.
